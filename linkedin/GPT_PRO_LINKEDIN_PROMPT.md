# Prompt For GPT Pro

Using this repository as context, write a polished LinkedIn post in my voice about how I structure software projects so AI coding agents can work productively with GitHub, AWS, and real production infrastructure.

Do not make it sound like AI hype. Make it sound like a practical engineering reflection from someone learning how to make AI agents reliable teammates.

Core message:

AI coding gets dramatically better when repository structure, cloud architecture, and agent workflow are treated as one integrated system.

Important ideas to include:

- GitHub is the source of truth; AWS is deployment state.
- AI agents need durable repo instructions, not just one-off prompts.
- Meaningful AI-authored changes should be committed, pushed, logged, checked, and deployed when relevant.
- Changelogs and architecture docs are not bureaucracy; they are context memory for future agents.
- AWS accounts and environments should be isolated so mistakes do not spill across projects.
- Infrastructure should be repeatable and encoded, not hand-wired.
- The browser should never receive AWS credentials; backend roles and presigned URLs are the safe pattern.
- A scalable app architecture can be simple: TypeScript frontend, backend API, CloudFront/S3, API Gateway/Lambda, DynamoDB/S3, and infrastructure-as-code.
- Sometimes the right move is to wipe an older architecture and rebuild it with clearer boundaries instead of carrying forward architectural drag.

Avoid:

- Overclaiming that AI replaces engineers.
- Sounding like a vendor announcement.
- Too many implementation details.
- Mentioning private emails, IDs, account numbers, tokens, or secrets.

Tone:

- Clear.
- Thoughtful.
- Technical.
- Practical.
- Confident but not grandiose.

Please produce:

1. One main LinkedIn post.
2. A shorter alternate version.
3. Three possible opening hooks.
